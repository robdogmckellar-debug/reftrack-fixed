# Phase 4, Chunk 2 — TypeScript and electron-vite foundation

## Scope

This chunk changes the development and production build foundation without redesigning screens or intentionally changing application behaviour.

Implemented:

- electron-vite 5 build pipeline
- Vite 7 renderer build
- Strict TypeScript project references
- Separate main, preload and renderer source folders
- Typed preload bridge
- Typed temporary IPC facade
- TypeScript performance instrumentation
- Production output under `out/main`, `out/preload` and `out/renderer`
- Development renderer loading through `ELECTRON_RENDERER_URL`
- TypeScript-aware ESLint configuration
- Type checking added to the standard verification command

Deferred intentionally:

- Main-process architecture replacement
- Domain-command IPC contract
- Atomic persistence
- Folder Cleaner safety replacement
- Preact renderer and design system
- Windows packaging hardening

## Source movement

The following legacy source files were moved into the approved process-oriented folders:

| Previous path                 | Current path                       |
| ----------------------------- | ---------------------------------- |
| `src/main.js`                 | `src/main/index.js`                |
| `src/performance-baseline.js` | `src/main/performance-baseline.ts` |
| `src/preload.js`              | `src/preload/index.ts`             |
| `src/index.html`              | `src/renderer/index.html`          |
| `src/app.js`                  | `src/renderer/app.js`              |
| `src/styles.css`              | `src/renderer/styles.css`          |

The main process and renderer remain JavaScript for now. Keeping them as legacy modules avoids combining build-system migration with high-risk behavioural refactoring.

## TypeScript policy

All new TypeScript code is compiled with strict settings, including:

- `strict`
- `noImplicitOverride`
- `noUncheckedIndexedAccess`
- `exactOptionalPropertyTypes`
- `useUnknownInCatchVariables`
- `isolatedModules`
- `verbatimModuleSyntax`

The root TypeScript project references separate main, preload and renderer configurations.

## Build outputs

`npm run build:app` produces:

```text
out/
├── main/
│   ├── index.js
│   └── performance-baseline.js
├── preload/
│   └── index.js
└── renderer/
    ├── index.html
    └── assets/
```

The renderer HTML references content-hashed CSS and JavaScript assets generated by Vite.

## Behaviour preservation

The current IPC channel names and renderer API method names are unchanged. A temporary `LegacyElectronApi` interface now documents that surface, but does not make it secure. The approved narrow command contract is still scheduled for Chunk 5.

The existing visuals, site data, statistics rules and daily-task behaviour are unchanged.

## Safety warning

The legacy Folder Cleaner remains unsafe in this chunk and can recursively delete non-image files. Keep it disabled. The image-only, non-recursive, Recycle Bin implementation is scheduled for Chunk 6.

## Windows verification

From a fresh extracted project folder:

```powershell
npm ci
npm run verify
npm start
```

Verify:

1. The application opens to the Dashboard.
2. All five screens visually match the previous chunk.
3. Copy Link still copies the expected text.
4. Success and Undo still behave as before.
5. Site Editor and Daily Tasks still load and save.
6. Statistics still opens and drills into months.
7. Closing and reopening the app retains data.

Do not test the Folder Cleaner with valued files.
